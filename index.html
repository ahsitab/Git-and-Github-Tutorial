Git & GitHub Visual Tutorial

Learn essential Git & GitHub commands step by step

Step 1: Install Git

Git is a version control system used to track changes in your code.
Download and install Git on your machine:
# Windows:
Download from git-scm.com/download/win

# Mac (using Homebrew):
brew install git

# Linux (Debian/Ubuntu):
sudo apt install git

Step 2: Initialize a Git Repository

Create a new project folder and initialize it as a Git repository:
mkdir my-project
cd my-project
git init
This creates a hidden .git folder that tracks your changes.

Step 3: Basic Git Workflow

The typical Git workflow involves these commands:
git status # Check file changes
git add . # Stage all changes
git commit -m "Your message" # Commit changes
- git status shows files that are modified.
- git add stages files for commit.
- git commit saves the staged snapshot with a message.

Step 4: Branching and Merging

Branches allow you to work on features separately without affecting main code.
git branch feature-1 # Create a new branch
git checkout feature-1 # Switch to branch
git merge feature-1 # Merge branch into current branch
Always merge your feature branch back into main when done.

Step 5: Connect to GitHub and Push Changes

Link your local repo to GitHub and upload your commits:
git remote add origin https://github.com/your-username/repo.git
git branch -M main
git push -u origin main
- git remote add connects your repo.
- git push uploads commits to GitHub.

Step 6: Collaborate Using Fork & Pull Requests

1. Fork a repo on GitHub.
2. Clone your fork locally.
3. Create a feature branch, make changes, commit, and push.
4. Open a Pull Request to merge changes upstream.
git clone https://github.com/your-username/forked-repo.git
git checkout -b feature-branch
git push origin feature-branch

Step 7: Undo Changes

Sometimes you want to undo changes locally or remove files from staging:
git checkout -- file.txt # Discard changes in working directory
git reset HEAD file.txt # Unstage a file
git reset --hard HEAD # Reset working directory and index to last commit
Use --hard reset carefully as it deletes local changes.

Step 8: Working with Remote Branches

Fetch and track branches from remotes:
git fetch origin # Fetch latest changes from remote
git checkout -b feature origin/feature # Create local branch tracking remote branch
git pull # Pull latest changes into current branch

Step 9: Rebasing Basics

Rebasing moves your commits on top of another base tip:
git checkout feature
git rebase main
This creates a cleaner linear history by replaying your changes onto main.

Step 10: Stashing Changes

Save unfinished changes temporarily to work on something else:
git stash # Save changes
git stash list # List stashed changes
git stash apply # Reapply changes

Step 11: Resolving Merge Conflicts

If two branches modify the same lines, Git stops and asks you to fix conflicts:
# Open conflicting files and edit
git add conflicted-file
git commit
Use your editor or tools like VS Code’s merge conflict UI for easier resolution.

Step 12: Tags and Releases

Mark important points (like v1.0) in history with tags:
git tag v1.0
git push origin v1.0
Tags can be used for release versions.

Step 13: Introduction to GitHub Actions

Automate workflows like testing and deployment with GitHub Actions:
Create a file .github/workflows/main.yml in your repo:
name: CI
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Run tests
      run: echo "Add your test commands here"
This runs tests automatically on every push.
